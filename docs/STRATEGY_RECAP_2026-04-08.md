# CelebBase Wellness — 하네스 완공 및 실전 구현(Phase 1) 가이드

## 요약
부트스트랩(Phase 0)을 통해 AI가 안전하게 달릴 수 있는 '하네스(규칙/검증망)' 구축을 완료했다. 이제 사용자(Grand Planner)가 목적지(MVP)를 설정하면, AI(Generator/Evaluator)가 하네스를 준수하며 자율 주행하는 단계로 진입한다. 도구(Gemini, Cursor 등) 간의 충돌을 방지하기 위해 '전략적 오케스트레이터' 중심의 워크플로우를 확립한다.

## 1. PGE 루프의 실전 운용 (역할 분담)

하네스가 채워진 지금, 개발은 아래와 같은 PGE(Planner-Generator-Evaluator) 루프로 돌아간다.

| 역할 | 담당 | 행동 지침 |
|------|------|-----------|
| **Grand Planner** | **사용자 (Human)** | 무엇을(What) 만들지 결정하고, AI가 제안한 전략(Strategy)을 승인한다. |
| **Strategic Orchestrator** | **Gemini CLI (AI)** | `spec.md`를 해석해 태스크를 쪼개고, 하네스 규칙을 지키며 코드를 짜고 스스로 검증한다. |
| **Harness (Referee)** | **Project Rules & CI** | `.claude/rules/`, `tests/`, `lint` 등이 AI가 규칙을 어기면 즉시 차단(Reject)한다. |

## 2. 도구별(Tools) 최적 활용 전략

도구들을 동시에 켜기보다, 작업의 성격에 따라 주도권을 배분한다.

- **Gemini CLI (주력 드라이버)**: 
    - **용도**: 복잡한 백엔드 로직, DB 스키마, 테스트 자동화, 인프라 설정 등 **'덩어리가 크고 규칙 준수가 중요한 작업'**.
    - **장점**: 프로젝트 전체 맥락 파악 및 도구(Shell, Scripts) 활용 능력이 가장 높음.
- **Cursor (조종석/리뷰)**: 
    - **용도**: 에이전트가 짠 코드 리뷰, 세밀한 UI/UX 수정, 사용자가 직접 코드를 고치고 싶을 때.
    - **장점**: 시각적 확인이 빠르고 코드 에디팅 경험이 뛰어남.
- **Claude Code (특수 요원)**: 
    - **용도**: 로직의 논리적 결함을 찾거나 고도의 코딩 정밀도가 필요할 때 교차 투입.

## 3. 개발 워크플로우 (Intent to Delivery)

1. **의도 전달 (Intent)**: 사용자가 자연어로 "이 기능을 구현해줘"라고 말함.
2. **태스크 구체화 (Planner)**: 에이전트가 `tasks.yaml`에 작업 단위를 추가하고 `Strategy`를 보고함.
3. **자율 주행 (Act & Validate)**: 승인 시 에이전트가 코딩(Generator)하고 스스로 테스트(Evaluator)하여 하네스 통과 확인.
4. **결과 보고 (Report)**: "하네스 규칙을 모두 준수하여 구현 완료했습니다."
5. **최종 승인 (Review)**: 사용자가 Cursor나 시뮬레이터로 시각적 검증 후 다음 단계로 이동.

## 4. 현재 상태 및 차기 과제

- **현재**: 모든 '길(Harness)'이 닦였고, 기초 스켈레톤 서비스가 구동 가능한 상태.
- **차기 과제: `IMPL-001` (기초 공사)**
    - **내용**: 실제 DB 테이블 생성(PostgreSQL) 및 전 서비스 공통 타입(Shared Types) 정의.
    - **목적**: 데이터 기반의 실제 비즈니스 로직을 올리기 위한 콘크리트 타설 작업.
